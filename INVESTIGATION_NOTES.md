# サンプル請求改善 - 調査メモ

## 認証・権限管理の仕組み

### 管理者ダッシュボード（/master/*）
- **認証系統**: `auth.ts` の `authRouter` → `users` テーブル（email/password認証）
- **auth.me**: `ctx.user` を返す → `User` 型（users テーブル）
  - `User.role` = "user" | "admin"
- **DashboardLayout.tsx**: `useAuth()` → `user.role !== 'admin'` → リダイレクト
- **ログイン時の自動昇格**: `auth.ts` 73-91行目
  - ログイン時に `staff` テーブルのメールと一致するか確認
  - 一致すれば `users.role` を `'admin'` に自動昇格
  - つまり **staffテーブルにメールが登録されている全スタッフ** がadminに自動昇格される

### 問題の本質
- staffテーブルにメールが登録されているスタッフは、ログイン時に自動的にadminに昇格される
- しかし、**一度もログインしていない**、または**staffテーブルにメールが登録されていない**スタッフはadminにならない
- HRスタッフが「アクセスできない」問題は以下の可能性：
  1. staffテーブルにメールが登録されていない
  2. usersテーブルに登録されていない（アカウントがない）
  3. ログインしたがstaffテーブルのメールと一致しなかった

### 結論
- 現状の仕組みでは、staffテーブルにメールが登録されている全スタッフがadminに自動昇格される
- 「一部のHRスタッフしかアクセスできない」問題は、staffテーブルのメール不一致が原因の可能性が高い
- ただし、ユーザーの要求は「全HRスタッフに開放」なので、DashboardLayoutの権限チェックを緩和する必要があるかもしれない
